使用轉換工具美化課程計畫

將 Word 匯出的 .htm 轉為美化後的 HTML

↗ 開啟轉換工具
  1. ① 開啟工具後，「設定基礎網址」欄位請保持空白，這樣連結才能以相對路徑指向 GitHub 上的子資料夾。
  2. ② 點擊「選擇檔案」，上傳從 Word「另存為 → 篩選網頁 (*.htm)」所產生的 .htm 檔案。
  3. ③ 選擇美化風格（推薦「專業藍」），填入或修改網頁標題後，點擊「套用並顯示」。
  4. ④ 預覽正確後，視需要用 A－ / A＋ 調整標題與內文字型大小。
  5. ⑤ 點擊「💾 下載處理後的網頁」，儲存 調整後的課程計畫首頁.html 到電腦中，備用。
⚠️ 重要：「設定基礎網址」請保留空白不填。若填入網址，連結會被轉換為 Google Apps Script 格式，而非相對路徑，導致 GitHub Pages 上連結失效。
2

整理所有要上傳的檔案

確認主頁 HTML 與各科資料夾都已備齊

請將以下所有項目放在同一個父資料夾中，方便一次選取上傳：

  • ✅ 主要首頁 HTML：步驟 1 下載的 調整後的課程計畫首頁.html
  • ✅ 各科課程計畫資料夾：Word 另存網頁時同時產生的資料夾（內含各科目 .html 及圖片等）。資料夾名稱必須與主頁中的超連結路徑完全一致。
建議的資料夾結構：
📁 上傳資料夾（自訂名稱）
  ├── 📄 調整後的課程計畫首頁.html
  ├── 📁 國語/
      ├── 114國語課程計畫.html
      └── 114國語課程計畫_files/
  ├── 📁 數學/
  ├── 📁 自然/
  └── 📁 ...（其他科目）
💡 提示：步驟 3 上傳時，選擇這個「上傳資料夾」即可一次完成，系統會自動保留所有子資料夾結構。
3

上傳至 GitHub 並自動發佈

填入 Token、選擇倉庫、選取檔案，一鍵完成

如何取得 GitHub Personal Access Token？
  1. 登入 GitHub，點右上角頭像 → Settings
  2. 左側最下方點 Developer settings
  3. 選 Personal access tokens → Tokens (classic)
  4. 點 Generate new token (classic)
  5. Note 欄輸入名稱（如：課程計畫上傳），Expiration 選擇有效期
  6. 勾選 repo 範圍（其下所有子項會自動勾選）
  7. 點 Generate token 並立即複製（只顯示一次！）
  8. 貼到上方輸入欄，點「驗證帳號」
⚠️ Token 等同密碼，請勿分享給他人。用完後可到 GitHub 設定頁面刪除。

注意：上傳至現有倉庫，同名檔案會被覆蓋更新。

點擊後選取「步驟 2 整理好的上傳資料夾」，系統會自動讀取資料夾內所有檔案（含子資料夾）。

💡 上傳時會保留完整資料夾結構，主頁 HTML 與各科資料夾都會放在倉庫根目錄，連結即可正常運作。
